## How it works

Cymbal Shops services communicate exclusively over [gRPC](https://grpc.io) using strongly-typed [Protocol Buffers](https://protobuf.dev) (proto3). The single source of truth for all API contracts is the proto file at `protos/demo.proto` in the repository root.

Each service exposes a well-defined gRPC interface. The frontend service acts as the gateway, translating HTTP requests from the browser into gRPC calls to the backend services. All other services communicate directly with each other via gRPC.

## Services

| Service | RPCs | Description |
|---|---|---|
| CartService | 3 | Manages per-user shopping cart state stored in Redis |
| CheckoutService | 1 | Orchestrates the full checkout flow across multiple services |
| ProductCatalogService | 3 | Serves product listings, lookups, and search |
| RecommendationService | 1 | Returns product recommendations based on the current cart |
| ShippingService | 2 | Provides shipping quotes and generates tracking IDs (mock) |
| CurrencyService | 2 | Lists supported currencies and converts monetary amounts |
| PaymentService | 1 | Processes credit card charges (mock) |
| EmailService | 1 | Sends order confirmation emails (mock) |
| AdService | 1 | Serves contextual advertisements based on page keywords |

## Generating client code

Use the `protoc` compiler with the appropriate language plugin to generate gRPC client stubs from `protos/demo.proto`.

<CodeGroup>

```bash Go
protoc \
  --go_out=. \
  --go_opt=paths=source_relative \
  --go-grpc_out=. \
  --go-grpc_opt=paths=source_relative \
  protos/demo.proto
```

```bash Python
python -m grpc_tools.protoc \
  -I. \
  --python_out=. \
  --grpc_python_out=. \
  protos/demo.proto
```

```bash Node.js
grpc_tools_node_protoc \
  --js_out=import_style=commonjs,binary:. \
  --grpc_out=grpc_js:. \
  protos/demo.proto
```

</CodeGroup>

The proto file uses the `hipstershop` package and the Go package path `github.com/GoogleCloudPlatform/microservices-demo/hipstershop`.
